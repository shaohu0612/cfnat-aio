# V2.1 IP 池管理优化 - 实施计划

## 任务列表

### 任务 1: 扫描后自动重建池 (5.1)
- **文件**: `internal/scanner/scanner.go`
- **修改**: `RunOnce` 阶段 5 入库后，添加阶段 6 调用 `s.lib.RebuildPools`
- **代码**:
  ```go
  // 阶段 6: 重建 IP 池
  s.setProgress("6/6 重建IP池", 0, 0, "")
  pf := s.cfgMgr.ProxyForward()
  s.lib.RebuildPools(pf.ActivePoolSize, pf.StandbyPoolRatio)
  logging.InfoTo("scanner", "  [6/6] IP 池重建完成")
  ```

### 任务 2: 批量入库优化 (5.2)
- **文件**: `internal/iplibrary/library.go`
- **修改**: 新增 `AddIPsBatch` 方法，批量插入 + 单次 reload
- **文件**: `internal/config/store_sqlite.go`
- **修改**: 新增 `UpsertIPsBatch` 方法，使用事务批量插入
- **修改 scanner.go**: `saveByRegion` 和 `saveByCMIN2Colo` 改为批量收集后统一入库

### 任务 3: 健康检查覆盖所有 IP (5.3)
- **文件**: `internal/proxy/manager.go` (或等效文件)
- **修改**: 健康检查循环中，对所有入库 IP（不仅是有连接的 IP）做主动探活
- **修改**: 降低样本门槛，基于探活结果判断隔离

### 任务 4: IP 淘汰算法升级 (5.4)
- **文件**: `internal/iplibrary/library.go`
- **修改**: 新增 `RemoveLowQualityIPs` 方法，基于综合质量分数淘汰
- **质量分数公式**: `100 * latencyScore * lossScore * speedScore * freshnessScore`
- **修改**: 在 `RebuildPools` 中集成淘汰逻辑，或作为独立后台任务

### 任务 5: WebUI 适配
- **文件**: `internal/webui/templates/index.html`
- **修改**: IP 库页面显示质量分数、健康状态、淘汰策略配置
- **修改**: 扫描进度增加"重建IP池"阶段显示

## 依赖关系
- 任务 2 依赖任务 1（需要 RebuildPools 调用批量优化后的方法）
- 任务 3 和 4 相对独立
- 任务 5 依赖所有后端任务

- 执行 `测试用例v2.md` 中 V2.1 相关测试用例
- 检查日志中"[6/6] IP 池重建完成"
- 检查数据库批量插入性能
- 检查 WebUI IP 库页面新字段
