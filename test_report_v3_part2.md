## 五、Docker 部署测试（DOCK-V3-001~DOCK-V3-005）

### 5.1 Dockerfile 优化

**问题**: Docker 构建时无法访问 Alpine 官方镜像源和国内镜像源

**解决方案**: 修改 Dockerfile 使用国内镜像源

```dockerfile
# Go 代理改为国内源
RUN go env -w GOPROXY=https://goproxy.cn,direct && \
    go mod download

# Alpine 镜像源改为国内源
RUN sed -i 's/dl-cdn.alpinelinux.org/mirrors.ustc.edu.cn/g' /etc/apk/repositories && \
    apk add --no-cache ca-certificates tzdata
```

### 5.2 构建结果

| 用例编号 | 测试场景 | 结果 | 原因 |
|----------|----------|------|------|
| DOCK-V3-001 | Docker 构建 | ❌ | 网络连接问题，无法访问 apk 镜像源 |
| DOCK-V3-002 | 挂载 ip.txt | ⏭️ | 构建失败，无法继续 |
| DOCK-V3-003 | 数据卷持久化 | ⏭️ | 构建失败，无法继续 |
| DOCK-V3-004 | IPv6 环境检测 | ⏭️ | 构建失败，无法继续 |
| DOCK-V3-005 | WebUI 可访问 | ⏭️ | 构建失败，无法继续 |

**备注**: Dockerfile 已优化，但当前网络环境无法访问 apk 镜像源。建议在网络通畅的环境下重新构建。
