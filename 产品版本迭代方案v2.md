## 2 多领域专家综合诊断报告

### 2.1 网络专家诊断 — CIDR 来源与抽样命中率

**根因定位**：扫描器 `loadCIDRs()` 从远程 URL 获取 Cloudflare 官方 CIDR 列表，为 `/13`、`/14`、`/15` 等大段。而用户本地 `ip.txt` 是经过筛选的 `/24` 精细段。

**命中率计算**：

| CIDR 段类型 | 主机位数 | 可用 IP 数 | 抽 100 个命中率 |
|------------|---------|-----------|----------------|
| `/24`（ip.txt） | 8 | 254 | ~39%（100/254） |
| `/13`（官方） | 19 | 524,288 | ~0.019%（100/524,288） |
| `/14`（官方） | 18 | 262,144 | ~0.038% |
| `/15`（官方） | 17 | 131,072 | ~0.076% |

**结论**：官方大段 CIDR 中绝大部分 IP 不在线或不可路由，抽样 100 个几乎必然全部 miss。

**代码缺陷**：`sampleCIDRs` 函数参数名为 `SamplesPer24`，暗示按 `/24` 抽样，但实际是对传入的每个 CIDR 整体抽样。对于 `/13` 大段，只抽 100 个，完全丧失 `/24` 抽样的意义。

### 2.2 算法专家诊断 — 测速与过滤阈值

**根因定位**：`MinSpeedMBps` 默认 `3.0`，`OnlyCMIN2` 默认 `true`，双层过滤导致可用 IP 几乎为零。

**阈值分析**：

用户截图显示的 IP 延迟为 251-262ms，按 TCP BBR 吞吐量估算：
- RTT ≈ 260ms
- 带宽延迟积限制下，单连接吞吐量受限于拥塞窗口
- 实际测速结果通常在 0.5-2.0 MB/s 之间
- **3.0 MB/s 阈值过滤掉了 90% 以上的可用 IP**

**speed.cloudflare.com 429 问题**：
- 测速并发限制为 5，但 429 退避策略不完善
- 项目 memory 已记录："Manual HTTP parsing in measureSpeed function causes silent failures when speed.cloudflare.com returns HTTP 429"
- 即使 IP 实际可达，429 也会导致测速失败

**CMIN2 白名单限制**：
- CMIN2 colo 白名单仅 15 个：HKG, SIN, NRT, KIX, LAX, SJC, SEA, FRA, AMS, LHR, TPE, ICN, MNL, BKK, MFM
- LAX 虽然在白名单中，但大部分美国 colo 不在白名单中
- 若扫描到其他美国 colo（如 DFW, ORD 等），会被直接丢弃

### 2.3 数据库专家诊断 — Region/Colo 映射与查询一致性

**根因定位**：`saveByRegion` 使用 `region := r.colo` 入库，但 `pickTarget` 使用地区配置的 `Code` 字段查询。

**映射关系分析**：

| 地区 | 配置 Code | 扫描到的 Colo | 入库 Region | 查询 Codes | 是否匹配 |
|------|-----------|---------------|-------------|-----------|---------|
| 香港 | HKG | HKG | HKG | [HKG] | 是 |
| 日本 | KIX,NRT,OKA,FUK | KIX | KIX | [KIX,NRT,OKA,FUK] | 是 |
| 新加坡 | SIN | SIN | SIN | [SIN] | 是 |
| 越南 | DAD,HAN,SGN | DAD | DAD | [DAD,HAN,SGN] | 是 |
| 美国 | DEN,DFW,...,LAX,...,TPA | LAX | LAX | [DEN,DFW,...,LAX,...,TPA] | 是（LAX 在列表中） |

**当前系统美国地区 Code 已包含 LAX，所以此问题在 V1.7 中已部分缓解。**

**但潜在风险**：如果扫描到不在地区 Code 列表中的 colo（如美国地区扫描到 YYZ 多伦多），入库 region=YYZ，查询时查不到，导致 IP 丢失。

### 2.4 调度专家诊断 — 负载均衡与健康检查

**根因定位**：健康检查只覆盖有代理连接记录的 IP，扫描器入库的新 IP 如果没有被代理使用过，永远不会被健康检查。

**健康检查逻辑缺陷**：

```go
// checkRegionHealth
metrics := m.metricsMgr.Get(entry.IP)
if metrics.GetSampleCount() < 5 {
    continue  // 样本不足，跳过检查
}
```

- 新入库 IP 的 `sampleCount = 0`
- 只有代理连接成功/失败时才会调用 `UpdateDelay`/`UpdateLoss`
- 如果新 IP 从未被代理选中（随机概率低），`sampleCount` 永远为 0
- 这些 IP 永远不会被健康检查，也不会被淘汰

**健康检查频率过高**：
- `healthCheckLoop` 使用 `time.NewTicker(time.Second)`，每秒执行一次
- `performHealthCheck` 对每个地区遍历所有 IP
- 当 IP 库有数百个 IP 时，每秒全量扫描造成不必要的 CPU 负载

**预热期死锁**：
- `isInWarmup` 检查 IP 是否在预热期内（默认 60 秒）
- 预热期内 IP 不被健康检查
- 但新 IP 也没有被代理优先使用的机制
- 导致新 IP 处于"不被检查也不被使用"的状态

### 2.5 信息系统专家诊断 — 架构层问题

**双层 IP 池未充分利用**：
- `RebuildPools` 只在配置热更新时调用
- 扫描器批量入库后不会自动重建
- 活跃池和备选池可能长期为空或过时

**缓存刷新性能问题**：
- `AddIP` 每次入库后调用 `reload()`，重建整个内存缓存
- 批量导入 100 个 IP 会触发 100 次全量 reload
- 时间复杂度 O(N^2)，N 为 IP 总数

**兜底池质量差**：
- `getFallbackCandidates` 从 `fallbackCIDRs`（大段 CIDR）中每 /24 抽 1 个
- 没有探活验证，直接使用未经验证的 IP
- 兜底成功率低

### 4.2 CIDR 抽样逻辑修复

**需求**：`SamplesPer24` 真正按 `/24` 粒度抽样，而不是按原始 CIDR 粒度。

**实现方案**：

```go
func (s *Scanner) sampleCIDRs(cidrs []string, samplesPer24, ipType int) []string {
    // 1. 将所有 CIDR 统一拆分为 /24 段
    var cidr24s []string
    for _, c := range cidrs {
        ones, _ := ipnet.Mask.Size()
        if ones >= 24 {
            cidr24s = append(cidr24s, c)
        } else {
            // 将 /n (n<24) 拆分为多个 /24
            cidr24s = append(cidr24s, splitTo24(c)...)
        }
    }
    
    // 2. 对每个 /24 抽样
    for _, c24 := range cidr24s {
        for i := 0; i < samplesPer24; i++ {
            ip := randomIPInCIDR(c24)
            out = append(out, ip)
        }
    }
}
```

**验收标准**：
- [ ] `/13` CIDR 正确拆分为 8192 个 `/24` 段
- [ ] 每个 `/24` 段抽样 `SamplesPer24` 个 IP
- [ ] 日志显示"X 个 /24 段，共 Y 个候选 IP"

### 4.3 测速策略优化

**需求**：降低默认阈值、支持延迟-only 模式、优化 429 处理。

**实现方案**：

### 4.4 扫描结果 region 映射修正

**需求**：入库时使用地区配置的 Name 作为 region，而不是 colo 代码。

**实现方案**：

```go
func (s *Scanner) saveByRegion(passed []speedResult, sc config.ScannerConfig) map[string]int {
    out := make(map[string]int)
    regions := s.cfgMgr.Regions()
    
    for _, r := range passed {
        if !r.ok || r.speedMbps < sc.MinSpeedMBps {
            continue
        }
        
        // 查找该 colo 属于哪个地区配置
        regionName := s.findRegionByColo(r.colo, regions)
        if regionName == "" {
            regionName = r.colo // 回退到 colo
        }
        
        err := s.lib.AddIP(r.ip, regionName, "auto", r.colo, r.speedMbps, r.latency, "scanner")
        if err == nil {
            out[regionName]++
        }
    }
    return out
}
```

**验收标准**：
- [ ] LAX IP 入库到 region="美国"，而不是 region="LAX"
- [ ] WebUI IP 库页面按地区名称分组显示

### 5.3 健康检查覆盖所有 IP

**需求**：健康检查覆盖所有入库 IP，不仅限于有代理连接记录的 IP。

```go
func (m *Manager) checkRegionHealth(r config.ProxyRegion) {
    codes := parseCodes(r.Code)
    
    for _, code := range codes {
        ips := m.lib.ListIPs(code)
        for _, entry := range ips {
            if m.isIsolated(entry.IP) {
                continue
            }
            
            // 对所有 IP 做主动探活
            ok, latency := m.probeIP(entry.IP)
            _ = m.store.MarkIPChecked(entry.IP, r.Name, ok, entry.SpeedMbps, latency)
            
            if !ok {
                m.metricsMgr.Get(entry.IP).UpdateLoss(true, alpha)
            } else {
                m.metricsMgr.Get(entry.IP).UpdateDelay(latency, alpha)
            }
            
            // 基于探活结果判断是否需要隔离
            metrics := m.metricsMgr.Get(entry.IP)

                lossRate := metrics.GetLossRate()
                if delay > float64(cfg.MaxDelayMs) || lossRate > cfg.MaxLossRate {
                    m.isolateIP(entry.IP, cfg.IsolationDuration)
                }
            }
        }
    }
}
```

### 5.4 IP 淘汰算法升级

**需求**：基于综合质量分数的淘汰策略，而不是简单的连续失败次数。

**实现方案**：

```go
func (l *Library) RemoveLowQualityIPs(threshold float64) (int, error) {
    allIPs := l.ListIPs("")
    var toRemove []string
    
    for _, entry := range allIPs {
        score := calculateQualityScore(entry)
        if score < threshold {
            toRemove = append(toRemove, entry.IP)
        }
    }
    
    for _, ip := range toRemove {
        l.store.DeleteIP(ip, entry.Region)
    }
    
    l.reload()
    return len(toRemove), nil
}

// 综合质量分数 = 延迟分 * 丢包分 * 速度分 * 新鲜度分
func calculateQualityScore(entry config.IPEntry) float64 {
    latencyScore := 100.0 / (entry.LatencyMs + 100.0)
    lossScore := (100.0 - entry.LossRate) / 100.0
    speedScore := entry.SpeedMbps / 10.0 // 10MB/s 满分
    if speedScore > 1.0 { speedScore = 1.0 }
    
    // 新鲜度：越久未检查分数越低
    freshnessScore := 1.0
    if entry.LastCheck != "" {
        lastCheck, _ := time.Parse(time.RFC3339, entry.LastCheck)
        hoursSince := time.Since(lastCheck).Hours()
        freshnessScore = math.Exp(-hoursSince / 24.0) // 24 小时衰减到 37%
    }
    
    return 100.0 * latencyScore * lossScore * speedScore * freshnessScore
}
```

## 6 V2.2 负载均衡升级

### 6.1 实时质量分数调度

**需求**：负载均衡基于实时质量分数，而非静态速度值。

**实现方案**：

```go
func (m *Manager) pickByQualityScore(codes []string, exclude map[string]bool) (string, error) {
    var candidates []struct {
        IP    string
        Score float64
    }
    
    for _, code := range codes {
        ips := m.lib.ListIPs(code)
        for _, entry := range ips {
            if exclude[entry.IP] || m.isIsolated(entry.IP) {
                continue
            }
            
            metrics := m.metricsMgr.Get(entry.IP)
            score := metrics.QualityScore()
            
            // 如果无样本，使用入库时的速度/延迟估算
            if metrics.GetSampleCount() == 0 {
                score = calculateQualityScore(entry)
            }
            
            candidates = append(candidates, struct{IP string; Score float64}{entry.IP, score})
        }
    }
    
    // 加权随机选择
    return weightedRandomSelect(candidates)
}
```

### 6.2 自适应阈值

**需求**：根据网络环境自适应调整延迟和速度阈值。

**实现方案**：

```go
func (m *Manager) adaptiveThresholds(codes []string) (maxDelayMs int, minSpeedMbps float64) {
    baseDelay := m.cfgMgr.ProxyForward().MaxDelayMs
    baseSpeed := m.cfgMgr.Scanner().MinSpeedMBps
    
    // 计算当前所有 IP 的 P50 延迟
    var delays []float64
    for _, code := range codes {
        for _, entry := range m.lib.ListIPs(code) {
            if entry.LatencyMs > 0 {
                delays = append(delays, entry.LatencyMs)
            }
        }
    }
    
    if len(delays) > 0 {
        p50 := percentile(delays, 50)
        p90 := percentile(delays, 90)
        
        // 自适应：阈值设为 P50 的 2 倍或 P90，取较小值
        adaptiveDelay := int(math.Min(p50*2, p90))
        if adaptiveDelay > baseDelay {
            maxDelayMs = adaptiveDelay
        }
    }
    
    return maxDelayMs, minSpeedMbps
}
```

## 7 V2.2.1 数据中心字典 + 地区选择升级

### 7.1 需求背景

**当前问题**：
- 添加地区页面为表单输入（Name、Code、Port、Enabled），Name 字段手工输入易冲突
- 用户对 Cloudflare 数据中心不熟悉，不知道有哪些 colo 代码可用
- Code 字段手工输入逗号分隔的列表，无可视化校验
- 地区名与 colo 的映射硬编码在配置里，新增/修改成本高

**目标**：
- 系统启动时自动从权威数据源拉取 Cloudflare 数据中心字典
- 数据中心字典中包含国家中文名、地区中文名、城市中文名、colo 代码、地理坐标
- 添加地区时通过下拉选择国家 -> 选择城市（多选） -> 自动生成 Code 字段
- 系统默认 5 个地区映射到字典中的数据，避免冲突

| VIVE Wi-Fi 国家代码 | ISO 国家代码（cca2）权威表 | https://business.vive.com/cn/support/dms/category_howto/wi-fi-country-codes.html |

**字段定义**（cloudflare-iata-full.json）：

```json
{
  "LAX": {
    "place": "Los Angeles, United States",
    "place_zh": "美国洛杉矶",
    "lat": 34.0522,
    "lng": -118.2437,
    "cca2": "US",
    "region": "North America"
  }
}
```

- `place`：英文城市，国家
- `place_zh`：中文国家名+城市名
- `cca2`：ISO 3166-1 alpha-2 国家代码
- `region`：Cloudflare 划定的大区

### 7.3 数据模型设计

新增数据库表：

```sql
-- Cloudflare 数据中心字典
CREATE TABLE IF NOT EXISTS cf_datacenter (
    iata         TEXT PRIMARY KEY,        -- colo 简码（IATA）
    place_en     TEXT NOT NULL,           -- 英文城市, 国家
    place_zh     TEXT NOT NULL,           -- 中文国家名+城市名
    country_zh   TEXT NOT NULL,           -- 国家中文名（如 "美国"）
    region_en    TEXT NOT NULL,           -- 大区英文（North America / Asia Pacific / ...）
    region_zh    TEXT NOT NULL,           -- 大区中文（北美洲 / 亚太地区 / ...）
    cca2         TEXT NOT NULL,           -- ISO 国家代码
    lat          REAL,                    -- 纬度
    lng          REAL,                    -- 经度
    enabled      INTEGER DEFAULT 1,       -- 是否启用（0/1）
    source       TEXT,                    -- 来源 URL
    updated_at   TEXT
);

CREATE INDEX idx_cf_dc_country ON cf_datacenter(country_zh);
CREATE INDEX idx_cf_dc_region  ON cf_datacenter(region_zh);

-- 数据中心字典元数据
CREATE TABLE IF NOT EXISTS cf_datacenter_meta (
    key          TEXT PRIMARY KEY,
    value        TEXT NOT NULL
);
-- 存储：source_url, last_sync_at, total_count
```

**Country 字段提取**：

从 `place_zh` 字段中按分隔符拆出国家名。`place_zh` 格式为 `{国家名}{城市名}`，已知规则：
- 多数为 `美国洛杉矶`、`日本东京`、`香港`（无分隔）
- 个别为 `吉布提市`（无国家前缀）
- 算法：取 en2zh.json 中 `place` 字段的"国家"部分长度，从 `place_zh` 截取对应前缀

```go
// extractCountry 提取国家中文名
// 优先从 en2zh.json 反查（如 "Los Angeles, United States" -> "美国洛杉矶" -> 国家长度 2 -> "美国"）
// 兜底使用 cca2 对照表
func extractCountry(place, placeZh, cca2 string) string {
    if en2zhEntry, ok := en2zhMap[place]; ok {
        if strings.HasPrefix(placeZh, en2zhEntry) {
            // 通过英文 place 拆分国家/城市
            parts := strings.Split(place, ", ")
            if len(parts) == 2 {
                countryEn := parts[1]
                if countryZh, ok := en2zhCountryMap[countryEn]; ok {
                    return countryZh
                }
            }
        }
    }
    // 兜底：通过 cca2 查询
    return cca2ToCountryZh[cca2]
}
```

| region (en) | region (zh) |

| North America | 北美洲 |
| South America | 南美洲 |
| Europe | 欧洲 |
| Africa | 非洲 |
| Asia Pacific | 亚太地区 |
| Oceania | 大洋洲 |

### 7.4 初始化流程

```go
// 在 config.Manager.init() 中追加步骤
func (m *Manager) initDC() error {
    // 1. 内置最新版本数据（编译期嵌入）
    embedded := embeddedCFDatacenters  // map[string]DCInfo

    // 2. 尝试远程拉取
    remote, err := fetchRemoteDCDict()
    if err == nil {
        embedded = remote
    } else {
        logging.WarnTo("dc", "远程数据中心字典拉取失败，使用内置: %v", err)
    }

    // 3. 写入数据库（去重 + 事务）
    return m.store.UpsertDatacenters(embedded)
}
```

**优先级**：
1. 用户手动同步按钮触发的远程拉取（最高）
2. 系统启动时静默拉取（失败则使用内置）
3. 内置的编译期数据（兜底）

**更新策略**：
- 仅在数据源版本号（commit hash）变化时更新
- 已存在的记录保留 `enabled` 状态（不覆盖用户配置）
- 新增记录默认 `enabled=1`
- 不存在的记录（数据源中已删除）保留数据库中的，不主动删除

### 7.8 数据迁移

老系统的 `regions` 表只有 `name`、`code` 字段，无 `country_zh`。V2.2.1 升级后：

```go
func (m *Manager) migrateRegions() error {
    regions, _ := m.store.LoadRegions()
    for _, r := range regions {
        if r.CountryZh == "" {
            // 通过 code 反查
            codes := parseCodes(r.Code)
            for _, code := range codes {
                if dc, ok := m.store.GetDatacenter(code); ok {
                    r.CountryZh = dc.CountryZh
                    break
                }
            }
        }
    }
    return m.store.SaveRegions(regions)
}
```
