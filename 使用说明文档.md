# CFNAT-AIO 使用说明

## 1 快速开始

### 1.1 部署

CFNAT-AIO 以单容器形式运行，依赖 Docker 和 Docker Compose。

```bash
git clone <repo>
cd cfnat-aio
docker compose up -d
```

容器启动后，WebUI 默认监听宿主机的 1234 端口。在浏览器中打开 `http://<服务器IP>:1234` 即可进入控制台。

### 1.2 默认端口映射

`docker-compose.yml` 中预置了以下端口映射：

| 用途 | 宿主机端口 | 容器端口 |
|------|-----------|----------|
| WebUI | 1234 | 1234 |
| 香港代理 | 2001 | 1001 |
| 美国代理 | 2002 | 1002 |
| 日本代理 | 2003 | 1003 |
| 新加坡代理 | 2004 | 1004 |
| 越南代理 | 2005 | 1005 |

首次启动时，系统自带 5 个默认地区：

| 地区名称 | 容器端口 | 对应 Colo 代码 |
|----------|----------|----------------|
| 香港 | 1001 | HKG |
| 美国 | 1002 | DEN,DFW,DTW,EWR,FSD,HNL,IAD,IAH,IND,JAX,LAS,LAX,MCI,MEM,MFE,MIA,MSP,OKC,OMA,ORD,ORF,PDX,PHL,PHX,PIT,RDU,RIC,SAN,SAT,SEA,SFO,SJC,SLC,SMF,STL,TLH,TPA |
| 日本 | 1003 | KIX,NRT,OKA,FUK |
| 新加坡 | 1004 | SIN |
| 越南 | 1005 | DAD,HAN,SGN |

所有默认地区默认启用。如果修改了 `docker-compose.yml` 中的端口映射，重启容器后新映射生效。

## 2 仪表盘

仪表盘是 WebUI 的首页，展示系统全局状态和实时日志。

### 2.1 统计卡片

页面顶部展示 4 个统计卡片：

| 卡片 | 说明 |
|------|------|
| 总 IP 数 | IP 库中所有地区的 IP 总数 |
| 活跃连接 | 当前所有地区的活跃代理连接数 |
| 隔离 IP | 被健康检查隔离的 IP 数量 |
| 预热 IP | 新入库正在预热保护期的 IP 数量 |

### 2.2 地区状态卡片

每个已启用的地区以卡片形式展示：

- **健康状态**：绿色圆点表示 healthy，黄色表示 degraded，红色表示 down
- **可用 IP**：该地区 IP 库中当前可用的 IP 数量。为 0 时卡片会提示 fallback 状态
- **当前 IP**：最近一次连接使用的上游 Cloudflare IP
- **监听状态**：绿色表示正在监听，灰色表示已停止
- **Colo**：该地区配置的 Cloudflare 数据中心代码
- **主池/备选池**：显示双层 IP 池的当前数量

### 2.3 实时日志

页面底部是日志流区域，通过 SSE 实时接收系统日志。日志按来源分类：

- `proxy`：代理转发日志（连接目标 IP、来源池等）
- `scanner`：扫描器日志（扫描进度、探活结果、测速数据）
- `webui`：WebUI 操作日志（导入探测、配置变更）
- `system`：系统级日志

日志区域自动滚动，鼠标悬停可暂停查看历史记录。

## 3 地区管理

地区管理模块控制多地区代理监听。每个地区对应一个独立的 SOCKS5/HTTP 代理端口。

### 3.1 添加地区

点击"添加地区"按钮，填写以下字段：

| 字段 | 填写示例 | 说明 |
|------|----------|------|
| 地区名称 | 日本 | 显示名称，支持中文 |
| Colo 代码 | KIX,NRT,OKA,FUK | 逗号分隔的 Cloudflare 数据中心简码 |
| 端口 | 1003 | 容器内部监听端口，需与 `docker-compose.yml` 中映射的容器端口一致 |
| 启用 | 勾选 | 是否立即启动监听 |
| Fallback | 勾选 | IP 库为空时是否自动使用全量 CF IP 池兜底 |

保存后系统自动校验端口冲突，通过后写入配置并启动 listener。

### 3.2 修改与删除

点击地区卡片上的编辑图标可修改配置。修改端口或 colo 代码后，系统会自动重启该地区的 listener。点击删除图标移除地区，对应 listener 立即关闭。

### 3.3 启用/停用

切换"启用"开关即可控制 listener 的启停，不需要删除地区配置。停用的地区不会显示在仪表盘状态卡片中。

## 4 CFNAT 配置

CFNAT 配置模块管理代理转发的基础参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| TLS 连接 | 开启 | 是否启用 TLS 加密连接到上游 Cloudflare |
| 随机选取 | 关闭 | 是否随机选择 IP 进行转发（关闭则使用加权随机） |
| IP 池大小 | 20 | 每个地区保留的可用 IP 数量 |
| 转发轮换数 | 1 | 同时用于转发的 IP 个数 |
| 测速时长 | 5 | 测速文件下载时长（秒） |
| 期望状态码 | 200 | HTTP 请求期望返回的状态码 |

## 5 CFData 扫描器

扫描任务对 Cloudflare IP 段进行系统性的探活和测速，自动将达标的 IP 纳入 IP 库。

### 5.1 扫描参数配置

在"扫描设置"标签页中配置以下参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 启用后台扫描 | 关闭 | 是否按间隔自动循环扫描 |
| 扫描间隔 | 60 分钟 | 两次扫描之间的等待时间 |
| 合格速度 | 3.0 MB/s | 入库的最低速度要求（建议根据网络环境调整） |
| IP 类型 | IPv4 | IPv4 或 IPv6（Docker 环境默认不支持 IPv6） |
| 端口 | 443 | 探活和测速使用的端口 |
| 每/24抽样 | 100 | 每个 /24 网段抽取的 IP 数量（1-255） |
| 延迟阈值 | 500 ms | 探活阶段延迟超限的 IP 直接丢弃 |
| 并发数 | 100 | TCP 探活的并发线程数 |
| 测速网址 | auto | 使用 Cloudflare 官方测速地址 |
| 只保留 CMIN2 节点 | 开启 | 只保留 CMIN2 白名单节点的 IP |

### 5.2 手动扫描

点击"立即扫描"按钮触发一次完整扫描。扫描过程分为 5 个阶段：加载 CIDR、抽样、探活、测速、入库。进度条和当前阶段名称会实时更新。扫描过程中可随时点击"停止扫描"中断任务。

### 5.3 IPv6 自动检测

系统启动时会自动检测 IPv6 连通性。如果用户配置了 IPv6 但环境不支持（如 Docker 容器），系统会自动切换到 IPv4 并记录警告日志。

### 5.4 扫描历史

扫描完成后，历史记录会保存在"扫描历史"列表中，包含：

- 启动时间和完成时间
- 扫描状态（成功/失败）
- 候选 IP 总数
- 最终达标入库数
- 详细统计 JSON（colo 分布等）

## 6 IP 库

IP 库存储经过验证的 CMIN2 优选 IP，按 colo 代码分库管理。

### 6.1 查看 IP 列表

IP 库页面以表格展示所有入库 IP，列包括：

| 列 | 说明 |
|----|------|
| IP | 目标 IP 地址 |
| Colo | Cloudflare 数据中心简码 |
| 地区 | 根据地区配置中的 colo 代码自动映射的地区名称 |
| 速度 | 测速下载速度（MB/s） |
| TCP延迟 | TCP 连接延迟（ms） |
| 来源 | auto（扫描器）、import（批量导入）、manual（手动添加） |
| 状态 | normal（正常）/ isolated（隔离）/ warming（预热中） |
| 池 | active（主池）/ standby（备选池）/ none（无） |
| 连接数 | 当前活跃连接数 |
| EWMA 延迟 | 指数加权平均延迟（实时监控） |
| EWMA 丢包率 | 指数加权平均丢包率（实时监控） |
| 权重 | 基于速度和延迟计算的权重值 |
| 失败次数 | 连续健康检查失败计数 |

### 6.2 筛选

- **地区筛选**：下拉列表显示地区管理中配置的地区名称（如"日本"、"美国"），选择后自动匹配该地区所有 colo 代码对应的 IP
- **来源筛选**：手动 / 自动扫描 / 批量导入
- **状态筛选**：全部 / 正常 / 隔离 / 预热中
- **池筛选**：全部 / 主池 / 备用池 / 无

### 6.3 手动添加 IP

在"手动添加"区域输入：

- IP 地址
- Colo 代码（如 KIX）
- 速度（MB/s）
- TCP 延迟（ms）

手动添加的 IP 不经过验证，直接入库。适合从外部渠道（论坛、社群）获取的已知可用 IP。

### 6.4 批量导入探测

批量导入探测用于快速将外部 IP 来源（如 FOFA 查询结果、CF-Workers 脚本输出）纳入 IP 库：

1. 将 IP 列表粘贴到文本框，支持以下格式：
   - 纯 IP：`104.16.1.1`
   - 带端口：`104.16.1.1:443`
   - 带注释：`104.16.1.1#日本节点`
2. 选择是否"自动入库"（达标 IP 自动写入 IP 库）
3. 点击"开始探测"

系统执行 6 步处理：解析→去重→并发探活→CMIN2 筛选→测速→入库。探测结果分为三类展示：CMIN2 达标、非 CMIN2、探测失败。

### 6.5 重建 IP 池

点击"重建 IP 池"按钮可手动触发双层 IP 池的重建，系统会根据当前配置的主池容量和备选池比例重新分配 IP。

### 6.6 删除 IP

点击表格行末的删除按钮，该 IP 会从 IP 库中移除，不再参与代理转发。支持批量选择删除。

## 8 系统设置

### 8.1 通用设置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| WebUI 端口 | 1234 | HTTP 服务监听端口，修改后需重启容器 |
| 数据目录 | /data | SQLite 数据库和日志存放路径 |
| 日志级别 | info | debug 会输出更多扫描细节 |
| API Token | 空 | 可选的 API 鉴权令牌 |

### 8.2 代理转发高级参数

展开"代理转发参数（V1.1+）"可配置以下高级参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 最大重试次数 | 3 | 连接失败后的重试次数（0-5） |
| 负载均衡策略 | 最少连接数 | 纯随机 / 最少连接数 / 加权随机 |
| EWMA 样本窗口 | 50 | 质量指标平滑窗口大小（20-200） |
| 健康检查间隔 | 120 | 自动健康检查间隔（30-600秒） |
| 最大延迟阈值 | 500 | IP 隔离延迟阈值（100-2000ms） |
| 最大丢包率 | 10 | IP 隔离丢包率阈值（1-50%） |
| 隔离时长 | 300 | IP 隔离后保留时间（60-900秒） |
| 预热保护期 | 60 | 新 IP 入库后免隔离时间（0-300秒） |
| 连接亲和性 | 关闭 | 同一客户端短时间内复用同一 IP |
| 亲和性 TTL | 15 | 亲和性保持时间（5-60秒） |
| 主池容量 | 20 | 活跃 IP 池大小（5-100） |
| 备选池比例 | 0.5 | 备选池容量为主池的比例（0.2-1.0） |

所有设置变更即时保存到 SQLite，大部分参数无需重启即可生效（WebUI 端口除外）。

## 9 客户端配置

CFNAT-AIO 提供 SOCKS5 和 HTTP CONNECT 两种代理协议，客户端连接各地区对应的端口即可使用。

### 9.1 qBittorrent

在 qBittorrent 的"连接"设置中：

- 代理类型：SOCKS5
- 主机：`192.168.7.4`（替换为你的服务器 IP）
- 端口：`2001`（香港）、`2002`（美国）、`2003`（日本）、`2004`（新加坡）或 `2005`（越南）
- 勾选"对连接使用代理"

### 9.2 v2rayN

在 v2rayN 中添加 SOCKS5 服务器：

- 地址：`127.0.0.1`（如果 v2rayN 与 CFNAT-AIO 在同一台机器）或服务器 IP
- 端口：`2003`（日本）或 `2004`（新加坡）等
- 协议：SOCKS5
- 无需认证

### 9.3 其他客户端

任何支持 SOCKS5 或 HTTP CONNECT 代理的客户端均可使用。连接对应地区的端口后，流量会被转发到该地区 CMIN2 IP 库中的随机 IP。如果 IP 库为空且启用了 fallback，则会使用全量 Cloudflare IP 池中的随机 IP。

## 10 常见问题

**Q：仪表盘显示可用 IP 为 0，但 IP 库页面有很多 IP**

A：检查地区配置中的 Colo 代码是否与 IP 库中的 colo 匹配。例如，地区配置为 `KIX,NRT` 但 IP 库中的 IP 的 colo 是 `OKA`，则不会被统计到该地区的可用 IP 中。

**Q：v2rayN 测试代理显示"无法连接到代理服务器"**

A：确认 docker-compose 中正确映射了对应端口（如 2003→1003）。检查防火墙是否放行了宿主机端口。在服务器上执行 `docker exec cfnat-aio ss -tlnp` 确认容器内端口正在监听。

**Q：扫描任务总是 0 个 IP 通过**

A：检查以下几点：
1. 测速阈值是否设得过高（默认 3.0 MB/s），建议改为 0.5 MB/s 测试
2. 检查服务器网络是否能正常访问外网（Cloudflare IP 和测速服务器）
3. IP 类型是否正确，Docker 环境默认不支持 IPv6

**Q：IP 库中的延迟为什么很多都小于 1ms**

A：延迟是服务器到 Cloudflare 节点的 TCP 连接延迟，不是用户端到端延迟。如果服务器在日本机房，连接 KIX 节点延迟确实可能低于 1ms，这是正常的。

**Q：修改地区配置后客户端需要重新配置吗**

A：如果修改了端口，客户端需要更新代理端口。如果仅修改 colo 代码或启用/停用，客户端配置无需变更。

**Q：如何备份数据**

A：SQLite 数据库文件位于容器的 `/data/cfnat-aio.db`，通过 Docker volume 映射到宿主机。直接备份宿主机上的数据库文件即可。重新部署时将该文件放回原位置，所有配置和 IP 库数据都会恢复。

**Q：什么是双层 IP 池**

A：双层 IP 池分为主池和备选池：
- 主池：活跃转发 IP，容量可配置（默认 20），按速度降序排列
- 备选池：待命补充 IP，当主池 IP 被隔离时自动从备选池提升最优 IP

**Q：连接亲和性有什么作用**

A：连接亲和性让同一客户端短时间内复用同一后端 IP，可以减少 TLS 握手开销，提高连接稳定性。适合需要保持会话一致性的场景。

**Q：EWMA 延迟和丢包率是什么**

A：EWMA（指数加权移动平均）是一种实时监控指标，通过给近期数据更高权重来平滑波动，能够快速反映 IP 的实时质量变化，用于加权随机选取和健康检查隔离。
