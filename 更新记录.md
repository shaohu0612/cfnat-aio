# CFNAT-AIO 程序更新记录

## 概述

CFNAT-AIO 是一款集成代理转发、IP 扫描和 WebUI 控制台的 Cloudflare CMIN2 优选 IP 工具。本文档记录所有版本的功能更新、技术变更和 bug 修复。

## 更新记录

### 2026-07-21 — V2.0 扫描引擎问题诊断与迭代方案

**问题**：其他 CFnat 客户端能搜索到大量 LAX 可用 IP，但当前系统 WebUI 中 IP 库显示为空。

**多领域专家综合诊断结论**：

| 领域 | 根因 | 影响 |
|------|------|------|
| 网络 | CIDR 来源使用 Cloudflare 官方大段（/13,/14），而非本地精细 ip.txt（/24） | 抽样命中率 <0.1% |
| 算法 | MinSpeedMBps=3.0 过高，OnlyCMIN2=true 限制过严 | 过滤掉 90%+ 可用 IP |
| 数据库 | 入库 region=colo，查询用地区 Code，存在映射不一致风险 | 部分 IP 查询不到 |
| 调度 | 健康检查只覆盖有代理连接记录的 IP，新入库 IP 永远不被检查 | 覆盖率 20% |
| 架构 | 双层 IP 池扫描后不自重建，缓存刷新 O(N^2) | 池长期为空 |

### 2026-07-20 — V1.7 完整版本

**更新内容**：一次性完成 V1.1 ~ V1.7 所有功能迭代，并进行前端 UI 全面升级。

#### 一、代理转发功能（V1.1 ~ V1.7）

##### V1.1 — 基础重试与故障转移
- **功能**：连接上游失败后自动重试，最多 N 次（可配置，默认 3 次）
- **配置项**：最大重试次数（0-5）
- **文件**：`internal/proxy/manager.go` — `handleConn()` 函数

##### V1.2 — 最少连接数负载均衡
- **功能**：从 IP 池中随机取两个候选，选择连接数更少者
- **配置项**：负载均衡策略（纯随机/最少连接数/加权随机）
- **文件**：`internal/proxy/manager.go` — `connCounts sync.Map`、`pickTarget()` 函数

##### V1.3 — 转发质量实时监控（EWMA）
- **功能**：使用指数加权移动平均（EWMA）实时监控每个 IP 的延迟和丢包率
- **配置项**：EWMA 样本窗口（20-200）
- **文件**：`internal/proxy/metrics.go` — `IPMetrics` 结构体

##### V1.4 — 主动淘汰与自动健康检查
- **功能**：后台定期健康检查，质量劣化 IP 先隔离后移除，新 IP 有预热保护期
- **配置项**：健康检查间隔（30-600s）、最大延迟阈值（100-2000ms）、最大丢包率（1-50%）、隔离时长（60-900s）、预热保护期（0-300s）
- **文件**：`internal/proxy/manager.go` — `healthCheckLoop` goroutine

##### V1.5 — 加权随机选取
- **功能**：根据 IP 速度和延迟计算权重，在候选池中加权随机选取
- **权重公式**：`weight = speed_mbps / (1.0 + ewma_delay_ms / 100.0)`

##### V1.6 — 连接亲和性（Sticky Sessions）
- **功能**：同一客户端短时间内复用同一后端 IP，减少 TLS 握手开销
- **配置项**：连接亲和性开关、亲和性 TTL（5-60s）
- **文件**：`internal/proxy/sticky.go` — `StickySlot` 机制

##### V1.7 — 双层 IP 池
- **功能**：将 IP 库拆分为主池（活跃转发）和备选池（待命补充），主池淘汰时自动从备选池提升
- **配置项**：主池容量（5-100，默认 20）、备选池比例（0.2-1.0，默认 0.5）
- **文件**：`internal/iplibrary/library.go` — 双层池结构

#### 二、WebUI 前端升级

##### 设计系统
- 深色主题全面升级，使用 CSS 变量驱动
- 玻璃质感卡片设计
- 渐变效果和发光边框
- 统一的颜色、间距、字体规范

##### 页面重写（7个页面）
- **仪表盘**：统计卡片（4列）、地区卡片（2列）、健康状态指示灯、双 IP 池显示
- **地区管理**：添加地区表单、地区列表表格
- **CFNAT 配置**：代理转发参数配置卡片
- **CFData**：扫描进度面板、扫描配置卡片、扫描历史表格
- **IP 库**：手动添加、批量导入探测 CMIN2、过滤搜索、导出功能、状态/池/权重列
- **日志**：实时日志流、来源/级别筛选
- **设置**：通用设置 + 可折叠的代理转发高级参数

#### 三、API 数据层增强

##### IP 库 API 扩展（`/api/ips`）
新增返回字段：
- `status` — IP 状态（normal/isolated/warming）
- `pool` — 所属池（active/standby/none）
- `conn_count` — 当前活跃连接数
- `ewma_delay_ms` — EWMA 平均延迟
- `ewma_loss_rate` — EWMA 丢包率
- `weight` — 权重值

##### 代理状态 API 扩展（`/api/proxy/status`）
新增返回字段：
- `sticky_count` — 活跃亲和性槽位数
- `active_pool_count` — 主池 IP 数量
- `standby_pool_count` — 备选池 IP 数量
- `health_status` — 健康状态（healthy/degraded/down）

##### 新增 API
- `POST /api/ips/rebuild-pools` — 手动重建双层 IP 池

#### 四、技术实现细节

##### 后端架构
- 纯 Go 实现，无 CGO 依赖
- SQLite 数据库持久化（`modernc.org/sqlite`）
- 并发安全：`sync.Map` + `atomic.Int32` 无锁计数
- 定时任务：健康检查循环、亲和性槽位清理

##### 前端架构
- 单文件 Vue.js 应用（`internal/webui/templates/index.html`）
- Naive UI 组件库
- 响应式布局，自适应宽度

##### Docker 部署
- 多阶段构建，最小运行时镜像
- Alpine 3.20 基础镜像
- 单二进制文件部署

## 配置参数汇总

### 代理转发配置（settings tab → 代理转发折叠区）

| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| 最大重试次数 | 0-5 | 3 | 连接失败后的重试次数 |
| 负载均衡策略 | 下拉选择 | 最少连接数 | 纯随机/最少连接数/加权随机 |
| EWMA 样本窗口 | 20-200 | 50 | 质量指标平滑窗口大小 |
| 健康检查间隔 | 30-600s | 120 | 自动健康检查间隔 |
| 最大延迟阈值 | 100-2000ms | 500 | IP 隔离延迟阈值 |
| 最大丢包率 | 1-50% | 10 | IP 隔离丢包率阈值 |
| 隔离时长 | 60-900s | 300 | IP 隔离后保留时间 |
| 预热保护期 | 0-300s | 60 | 新 IP 入库后免隔离时间 |
| 连接亲和性 | 开关 | 关闭 | 是否启用连接亲和性 |
| 亲和性 TTL | 5-60s | 15 | 亲和性保持时间 |
| 主池容量 | 5-100 | 20 | 活跃 IP 池大小 |
| 备选池比例 | 0.2-1.0 | 0.5 | 备选池容量为主池的比例 |
